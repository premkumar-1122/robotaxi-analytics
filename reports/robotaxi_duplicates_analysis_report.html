Key findings
Narratives

Changed in 26 of 28 groups

Most narrative changes are structural — original submissions were fully redacted ("CONFIDENTIAL BUSINESS INFORMATION") and updated versions added de-identified summaries. This is a deliberate PII-scrubbing process, not an error. The narrative content itself did not materially change in these cases.

Severity

Upgraded in 10 groups (36%)

Severity escalations were predominantly found in Waymo filings (4 of 5 groups) and Avride (3 of 5). The most common upgrade was from "Property Damage / No Injury" to "Minor W/O Hospitalization." Notably, one Waymo report (30270-11849) escalated to "Unknown" — meaning injury status remained unresolved even at time of re-filing.

Dates

Incident date unchanged in 27 of 28 groups

Only one group (Avride 31101-14647) had an incident date discrepancy — March vs April 2026. Submission dates varied widely, with some Tesla groups filing up to 5 months apart (Aug 2025 → Dec 2025 → Mar 2026), indicating a pattern of staged re-submissions rather than one-time corrections.

Completeness

All rows equally complete (score 20–21)

No row is strictly more complete than its pair — field-fill counts are identical within every group. The "richer" version is determined by narrative quality: the latest submission with a de-identified summary is always more informative than the initial redacted placeholder, but structurally both rows carry the same columns.

Notable anomalies

Severity inconsistency — Tesla 13781-11375 (4 rows)

This is the only ID with 4 duplicate filings. Two rows carry "Property Damage / No Injured" while a third carries "Minor W/ Hospitalization." The fourth row (Mar 2026 submission) reverts the narrative to a de-identified summary without resolving the severity conflict. The authoritative severity is ambiguous — deduplicate with caution.

Severity inconsistency — Avride 31101-13612 (3 rows)

Three unique severity values appear across the same incident: "Minor W/O Hospitalization", "Property Damage / No Injured", and "No Injured Reported" — all sharing the same submission date (Jan 2026). This may indicate simultaneous corrective re-filings with a conflicting field, or a data entry error at source.

Directional error — Zoox 30610-11847

Both filings share the same submission date, yet the narrative contradicts itself on vehicle direction: one says "southbound," the other says "eastbound." This is a factual correction embedded within an otherwise identical record — a strong signal that the later row should be treated as canonical.

System type correction — Waymo 30270-14096

The updated filing explicitly states that the automation system was reclassified from ADAS to ADS. This is a material regulatory distinction — ADAS and ADS fall under different reporting obligations. If this field is used for filtering, the original row will misclassify this incident.

PII removal pattern — all Tesla groups (15 IDs)

Every Tesla group follows an identical pattern: initial submission fully redacted → subsequent submission(s) contain a de-identified summary beginning "This updated report does not report a new incident or make any material changes..." This is a systematic, entity-wide compliance workflow, not ad-hoc corrections. Safe to deduplicate by always keeping the latest submission date row for Tesla.
